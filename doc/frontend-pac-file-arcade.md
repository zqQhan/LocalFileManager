# Pac-File Arcade 前端说明

`desktop` 模块已重构为复古街机游戏风 JavaFX 客户端。界面重点保持可用性：目录浏览、搜索、索引、导出、重复文件扫描、统计面板和文件右键操作都保留，文案改为直白功能名。

## 当前前端能力

- 目录导航：支持后退、前进、上一级、刷新和打开目录。
- 快速访问：只保留 Home 与 Documents，避免左侧路径过长。
- 搜索：支持文件名、文件内容、全部、正则、扩展名过滤。
- 索引：同步索引与 Kafka 异步索引仍调用后端真实接口。
- 导出：导出最近一次搜索结果为 CSV，并在弹窗中预览。
- 统计：展示后端 `/api/stats/dashboard` 返回的文件数量、大小、哈希和重复项。
- 加载动画：状态栏用吃豆人吃豆子的动画反馈长耗时操作。

## 后端限制说明

- 标签：后端当前只有标签 CRUD，没有提供“文件绑定标签”的 REST API。前端不伪造保存，只提示这是后端功能缺口。
- 索引：后端 `IndexService` 使用 `Files.walk`，遇到 Windows 受保护目录如 `Documents\\My Music` 会让整个索引失败。前端只展示明确错误，不修改后端。
